# 工作区文件说明 (Workspace Documentation)

这个文件夹定义了你的 AI 助手的“灵魂”和“认知”。这些文件是 Agent 在每次启动或对话时必须读取的核心上下文。

## 各文件用途与更新时机

### 1. 核心身份与规则
*   **`AGENTS.md`** (🔴 **最重要 / Critical**)
    *   **用途**: Agent 的 **行动纲领 (Manifesto)**。它定义了工作方式、内存管理规则、安全边界（什么能做，什么要先问），以及在群聊中的行为规范。
    *   **更新时机**: 当你想改变 AI 的根本行为模式时（例如：“晚上11点后不要主动发消息”或“在群聊里保持高冷”）。

*   **`SOUL.md`**
    *   **用途**: Agent 的 **性格内核 (Personality)**。它决定了 AI 的语气 (Vibe)、价值观和核心原则（例如：“要真诚帮助，不要虚伪客套”）。
    *   **更新时机**: 当你觉得 AI 的性格不对味，或者想让它扮演特定角色（比如变得更幽默、更严谨）时。

*   **`IDENTITY.md`**
    *   **用途**: Agent 的 **自我认知 (Self-Image)**。
        *   **包含内容**: 它的名字、物种设定（是机器人？还是赛博幽灵？）、代表它的 Emoji 签名等。
        *   **为什么需要**: 让它知道“我是谁”，而不是一个通用的 ChatGPT。
    *   **更新时机**: 刚初始化时，或者你想给它改名/换人设时。

### 2. 外部世界认知
*   **`USER.md`**
    *   **用途**: Agent 对 **你 (用户)** 的认知。记录你的名字、称呼、时区，以及它学到的关于你的偏好和背景（例如：“是个二次元”、“Python 开发者”、“喜欢《艾尔登法环》”）。
    *   **更新时机**: 随着你们的相处，AI 会自动（或由你手动）把学到的关于你的新知识写进去，以便长久记住。

*   **`TOOLS.md`**
    *   **用途**: **本地环境配置 (Local Config)**。记录你物理环境特有的东西。
        *   **例子**: “客厅摄像头叫 `cam_01`”、“TTS 语音用 `Nova`”、“家里服务器 IP 是 `192.168.1.5`”。
    *   **更新时机**: 当你的物理环境、设备名称或特定的工具参数发生变化时。

### 3. 运行机制
*   **`HEARTBEAT.md`**
    *   **用途**: **心跳/定时任务 (Cron-like)**。如果这个文件不为空，AI 会定期（心跳）检查这里的指令。
        *   **例子**: “每天早上检查日历”、“每 30 分钟看一次未读邮件”。
    *   **更新时机**: 当你需要 AI 主动定期关注某些事情时。

*   **`BOOTSTRAP.md`**
    *   **用途**: **出生证明 / 初始化引导脚本**。
        *   **作用**: 这是 Agent **第一次** 醒来时读的剧本。它告诉 Agent 第一句话该说什么，怎么跟你“破冰”，怎么引导你设置它的名字。
    *   **更新时机**: **仅在创建新 Agent 时存在**。通常在第一次对话完成后，Agent 就会把它删掉，因为它已经“出生”了，不再需要出生证明。

---
**一句话总结：**
*   **`AGENTS.md` / `SOUL.md` / `IDENTITY.md`** = AI 是谁 (怎么做事，什么性格，叫什么)。
*   **`USER.md`** = 你是谁。
*   **`TOOLS.md` / `HEARTBEAT.md`** = 它的工作环境和任务。
*   **`BOOTSTRAP.md`** = 它的出生指引 (用完即弃)。
